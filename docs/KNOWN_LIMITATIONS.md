# Known Limitations

- The default local MLX path now uses a repo-managed MLX conversion of the official `Qwen/Qwen3.5-35B-A3B` base model. It is locally downloaded and validated, but the MLX artifact itself is a conversion repo rather than an official Qwen-hosted MLX release.
- Qwen thinking is left enabled on the MLX path and EDEN surfaces it in a dedicated panel. When the first generation spends the whole token budget on reasoning, EDEN uses one bounded answer-only completion pass to obtain the final operator-facing answer.
- `adam_auto` on the MLX path currently falls back to `runtime_auto`. This fallback is visible and persisted; there is no hidden Adam prepass in v1.2.
- The session-start `debug` toggle is currently a persisted/session-visible flag only. It shows up in profile/runtime status surfaces, but it does not yet change MLX sampling, retrieval policy, or prompt assembly by itself.
- The prompt-budget ticker is an EDEN-side working estimate. It does not guarantee the true maximum usable context of every future model build.
- Exact token counting is only available when a tokenizer-backed counter is available. MLX preview mode before model/tokenizer load can still be heuristic.
- Geometry diagnostics are computed from graph topology, co-occurrence structure, active-set recurrence, and temporal traces. They do not inspect internal latent activations from the model itself.
- On large persistent graphs, observatory geometry falls back to sparse-safe approximations instead of dense spectral/PCA layouts. The outputs stay useful for operator diagnostics, but they are not mathematically identical to the small-graph exact path.
- Mirror, chirality, and translation-symmetry outputs are derived proxies. They are evidence-bearing diagnostics, but they are not proof of metaphysical “sacred geometry.”
- The browser observatory now runs as a checked-in React/Sigma/Three bundle, but it is still a bounded observability instrument rather than a full graph IDE with arbitrary drag-to-wire editing.
- Graph and basin rendering can fail in browsers or headless contexts without usable WebGL. The current bundle falls back to explicit renderer-unavailable notes plus text-access panels instead of inventing a visual surface or silently blanking the panel.
- The Tanakh tool-surface currently treats canonical Hebrew display as a DOM/CSS reader and keeps the Three scene explicitly derived. The render-validation harness produces oracle/HTML artifacts, but niqqud/cantillation placement still requires manual review before claiming fully validated 3D Hebrew text rendering.
- Geometry for editing previews is computed over EDEN's exported simple graph topology. Multiple relation types between the same node pair remain inspectable in provenance, but geometry itself is still evaluated on the collapsed neighborhood structure.
- Large-graph geometry exports cap per-memode local reports to a bounded, relevance-sorted subset so observatory/export remains responsive on this machine. The browser payload is intentionally not an exhaustive local-report dump for every memode.
- `current_active_set` geometry export is based on the latest persisted active set for the selected session, not the unsent live composer preview.
- `Resume Latest` restores the latest persisted session state, not an unsent draft from the composer.
- The merged runtime/event chyron is intentionally bounded and concise. It starts hidden as a bottom-up drawer (`F11` opens/closes it), and it exposes the latest runtime/event state without a full infinite-scroll event history in the prime screen.
- On compact terminals, EDEN now prioritizes transcript + composer legibility by hiding the right telemetry stack until the operator explicitly opens aperture/deeper surfaces. You can still work entirely from the keyboard, but simultaneous dialogue-and-telemetry visibility still wants a wider/taller terminal.
- Reply review is scoped to Adam's latest turn in the live session. It now lives inline in chat, and older turns keep their stored verdict labels on Adam's transcript cards rather than reopening as separate review flows.
- Conversation logs are written as markdown transcript artifacts for operator access, but they are not yet a richer replay/index surface with per-turn jump links, filters, or diff views.
- Conversation Atlas now projects saved sessions through one virtual folder path plus multi-value tags stored in `sessions.metadata_json`, but it is still a bounded terminal catalog rather than a full boolean-query knowledge manager or a physical filesystem mirror with duplicate transcript copies.
- The stock footer still clips some later function-key labels on narrow terminals. The compact composer hint now repeats the critical first-run keys, but the footer itself is not yet a fully responsive hotkey legend.
- The signal field now renders an orthographic memgraph bus based on active-set nodes, recent trace events, and ingest roots, but it is still an operator-facing abstraction rather than a literal visualization of hidden model activations or attention tensors.
- The `Typewriter Light` look is a persisted terminal appearance setting only. It changes the TUI palette and control-surface styling, but it does not alter session inference profiles, graph state, or any observability evidence surface.
- The terminal palette, chiaroscuro shading, dithered accents, and colored scrollbars are still constrained by Textual/Rich terminal rendering. The surface is now static and lighter on redraw work, but it is still not a pixel-true graphical material system.
- The prime screen no longer uses a constant whole-screen repaint loop, but live preview and memgraph animation still cost real work on large persistent-graph sessions. If the machine is under pressure, prefer `low_motion` and tighter inference budgets.
- Document ingest framing prompts are graph-indexed as persistent conditioning material, but they are not yet a separate ranked evidence class with their own dedicated retrieval controls.
- The repo-root `app.py` launcher reuses the repo-local `.venv`, but it does not repair a corrupted virtualenv by itself. If `.venv` was mixed across different base interpreters, rebuild it cleanly from one interpreter path.
- Legacy multi-experiment graph rows are not fully merged into one canonical meme/memode/edge space automatically. New and resumed sessions now route through one primary graph, but older duplicate graph partitions may still require a dedicated migration pass if exact historical union is needed.
- Retrieval remains lexical/graph-heuristic rather than embedding-based.
- The current hum is a bounded, minimal, read-only observability artifact derived from persisted active-set, feedback, and membrane-event surfaces. It is not a generation input, not prompt-injected, not membrane-consumed, and not equivalent to the active set, full history, full graph, basin summary, cluster summary, or observatory payloads.
- The new TUI/observatory hum and reasoning lenses are presentation surfaces only. They may show filtered visible reasoning excerpts, answer-beat reductions, and bounded hum entries / stats / motif counts, but they still do not expose or imply hidden chain-of-thought.
- Historical hum logs still exist and carry richer compressed motif behavior than the current bounded `hum.v1` artifact. The present build now renders a hum-like motif channel, but it still does not claim full historical parity or that the hum alone reconstructs persona.
- Known memode confidence is currently operator-supplied plus provenance-backed. There is not yet a separate learned confidence model for memode validity.
- The React observatory is now a browser-visible graph workbench with preview/commit/revert, compare, layout, filter, statistics, Data Lab, and runtime-trace surfaces, but it remains a bounded observability instrument rather than a general-purpose graph IDE with arbitrary import/plugins or unconstrained drag authoring.
- Tanakh runs are surfaced through dedicated sidecar artifacts and live read/run endpoints, but they are not first-class measurement events yet and do not participate in preview / commit / revert semantics.
- Static observatory exports must be HTTP-served. Direct `file://` opening is not a supported runtime path for the v1 bundle and, in the current Chromium proof, does not reach a truthful supported-static-ready state.
- Browser view presets live in `localStorage` only. They are origin-local, non-authoritative, and intentionally absent from the measurement ledger and graph payload planes.
- Browser-local layouts, style mappings, filter presets, ranking views, layout snapshots, and Data Lab sort/selection state live in `localStorage` only. They are origin-local, non-authoritative, and intentionally absent from the measurement ledger and graph payload planes.
- Layout execution and statistics stay bounded for this machine. Heavy graphs can trigger worker cancellation, progressive updates, or fallback to exported coordinates/capped metrics instead of pretending to offer unbounded Gephi-scale interactivity.
- The layout terrain browser is intentionally broader than the executable subset. Many algorithms are shown as reference/explanation entries, while only the worker-backed subset is runnable in-browser.
- The graph payload now exports layout-family ordering plus runnable-layout metadata, but the checked-in frontend terrain catalog still fills many reference-only algorithm entries so older exports and lean payloads render the same layout landscape.
- Export interoperability is currently one-way: Gephi-accepted downloads (`gexf`, `graphml`, `gdf`, `gml`, `graphviz dot`, `pajek net`, `netdraw vna`, `ucinet dl`, `tulip tlp`, `tgf`, node CSV, edge CSV) are supported, but arbitrary external graph import and plugin ecosystems remain out of scope.
- Format fidelity is not uniform across those downloads. Rich attribute-oriented formats such as `gexf`, `graphml`, and `gdf` preserve more EDEN metadata than sparse interchange formats such as `pajek net`, `tulip tlp`, and `tgf`, which are primarily topology-first.
- Gephi importer behavior is format-dependent. EDEN now emits semantic `export_label` fields for downloads, but `gexf` and `gdf` remain the highest-confidence choices when the operator wants Gephi's canonical Label column to reflect semantic content.
- The checked-in frontend bundle is treated as runtime contract surface. Runtime warns on stale build metadata, and CI should fail until `npm --prefix web/observatory run build` refreshes `build-meta.json`.
- The top action shelf observatory progress meter reports phase progress and accurate elapsed time only. It does not claim an ETA because server reuse, export cost, and host browser launch timing are not predictable enough to support a truthful remaining-time estimate.
